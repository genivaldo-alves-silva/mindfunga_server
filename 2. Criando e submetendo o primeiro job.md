# Criando e Submetendo o Primeiro Job

## 1. Enviando arquivos para o servidor

No teu terminal do teu computador, use:

```bash
scp -r /home/genivaldo/Downloads/infile.nex SEU_USUARIO@150.162.33.189:/home/SEU_USUARIO/filogenia/

# digite a senha do teu usuário quando requisitado
```
O arquivo de exemplo - infile.nex - está em /MrBayes Jobs.

## 2. Comandos iniciais já no servidor
- Comandos para verificar localização, criar arquivos e editar arquivos:

```bash
# verificar novamente
ls

# deve mostrar a presença da pasta filogenia e do arquivo .nex

# entrar na pasta criada
cd filogenia

sudo nano mrbayes.sub
```

## 3. Ao abrir o editor do mrbayes.sub, colo o conteúdo abaixo.

```ini
# mrbayes.sub
# Submissão de MrBayes via Condor

executable              = /usr/bin/mpirun
arguments               = --mca plm isolated --oversubscribe -np 8 /usr/local/bin/mb infile.nex

log                     = mrbayes.log
output                  = mrbayes.out
error                   = mrbayes.err

should_transfer_files   = NO

request_cpus            = 8
request_memory          = 6G
request_disk            = 7G

queue
```
Após colar o conteúdo, salve usando "Ctrl+X" em seguida "y" ou "s" e por fim "Enter".

## De volta ao terminal:
```bash
conda_submit mrbayes.sub

# verifique se o teu job foi submetido e se está rodando (em RUN)
condar_q

# Caso não apareça nada ou o job fique com HELD e não HOLD, verifique o arquivo de erro
cat mrbayes.err
# ou
nano mrbayes.err
```

## O arquivo mrbayes.sub pode ser criado e editado no teu computador e ser transferido junto com o infile.nex
```bash
#múltiplos | pedirá autenticação a cada arquivo
scp -r /home/genivaldo/Downloads/**{infile.nex,mrbayes.sub}** SEU_USUARIO@150.162.33.189:/home/SEU_USUARIO/filogenia/
```
